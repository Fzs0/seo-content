# 独立站通用 SEO 关键词评分与博客文章规划标准

## 1. 这几张 Excel 能不能作为标准？

可以，但要把它们当成一套标准包，而不是单独一张表。

当前标准包包含三层：

1. 关键词评分标准：判断一个关键词值不值得做、优先级高不高、应该由哪类页面承接。
2. 关键词到页面规划标准：判断关键词应该做分类页、集合页、产品页、解决方案页、博客文章、FAQ，还是暂时不做。
3. 博客文章规划标准：当关键词适合写文章时，进一步生成文章类型、父级商业页、Brief、内链、CTA 和复盘指标。

这套标准可以迁移到 ToB 和 ToC 独立站，但不能不改字段直接套。真正通用的是“判断顺序”和“评分逻辑”，不是某个行业样例词。

## 2. 通用总原则

关键词不是都拿来写文章。

独立站 SEO 的正确顺序是：

1. 先理解网站商业模型。
2. 再收集关键词。
3. 再看 SERP 判断页面类型。
4. 再给关键词打分。
5. 再分配页面。
6. 只有适合文章承接的词，才进入博客规划。
7. 文章必须通过内链服务商业页。
8. 发布后用 GSC/Semrush 复盘和更新。

博客不是终点，博客是辅助成交页面获得流量、信任和内部权重的工具。

## 3. ToB 与 ToC 的通用差异

| 维度 | ToB 独立站 | ToC 独立站 |
|---|---|---|
| 核心转化 | 询盘、报价、预约、下载资料 | 加购、下单、领取优惠、订阅 |
| 商业页 | 产品聚合页、解决方案页、行业页、案例页 | 分类页、集合页、产品页、对比页 |
| 博客作用 | 建立专业信任、解释方案、覆盖长尾问题 | 教育购买、比较选择、解决顾虑、促进下单 |
| 高价值词 | manufacturer、supplier、solution、custom、for industry | best、buy、review、vs、how to、benefits、for user group |
| CTA | Request a Quote、Contact Us、Download Catalog | Shop Now、View Collection、Add to Cart、Get Discount |
| 内容证据 | 案例、参数、标准、工厂、认证、工程经验 | 真实图片、测评、对比表、尺码、材质、用户场景 |

标准里所有“父级商业页”字段都可以迁移：

- ToB：父级商业页 = 产品聚合页 / 解决方案页 / 行业应用页 / 询盘页。
- ToC：父级商业页 = 分类页 / 集合页 / 商品页 / 购买指南落地页。

## 4. 关键词评分标准

关键词评分用于回答：这个词值不值得做？优先级是多少？应该用什么页面承接？

建议使用 100 分模型：

| 维度 | 权重 | 判断问题 |
|---|---:|---|
| 搜索意图匹配 | 20 | 这个词的 SERP 是否允许我们要做的页面类型排名？ |
| 商业价值 | 20 | 这个词是否能带来询盘、加购、下单或辅助成交？ |
| 需求强度 | 15 | 搜索量、CPC、GSC 展现是否证明有真实需求？ |
| 难度机会 | 15 | KD、竞品强度、站点阶段是否匹配？ |
| 页面可做性 | 15 | 我们是否有足够产品、内容、图片、数据、证据来做这个页面？ |
| 主题簇价值 | 10 | 是否能加入现有产品线/主题簇，并产生内链价值？ |
| 数据置信度 | 5 | 数据是否来自 Semrush/GSC/SERP 多方验证？ |

建议优先级：

| 总分 | 等级 | 动作 |
|---:|---|---|
| 75+ | P0 | 立即规划页面或文章 |
| 60-74 | P1 | 进入本批规划，但需要人工复核 |
| 45-59 | P2 | 低优先级，可能做 FAQ、段落或后续补充 |
| <45 | P3 | 暂不做，或合并到其他页面 |

## 5. 关键词分流标准

关键词评分后，必须分配页面类型。

| SERP/关键词信号 | 推荐承接页面 | 是否写博客 |
|---|---|---|
| buy、price、sale、shop、near me，前排多为商品/分类页 | 产品页、分类页、集合页 | 通常不写 |
| manufacturer、supplier、factory、custom | ToB 产品聚合页或解决方案页 | 可写辅助文章 |
| best、top、recommended | 购买指南、榜单文 | 适合写 |
| vs、comparison、alternative | 对比文 | 适合写 |
| how to、care、install、use、clean | 教程文、FAQ | 适合写 |
| benefits、good for、why | 价值解释文 | 适合写 |
| size、material、standard、specification | 标准/规格文 | 适合写 |
| what is、definition | FAQ 或商业页段落 | 多数不单独写 |
| 前排都是大站百科/论坛/品牌页 | 拆成长尾或暂缓 | 谨慎写 |

硬规则：

1. SERP 文章占比低于 4/10，不建议单独写博客。
2. 没有父级商业页，不建议单独写博客。
3. 与产品/服务弱相关，不建议写独立文章。
4. 只能写定义、无法提供证据和差异化，不建议写独立文章。

## 6. 博客文章规划标准

只有通过文章筛选的关键词，才进入博客规划。

博客规划不是只写标题，而是要输出完整 Brief：

| 模块 | 必须回答的问题 |
|---|---|
| 主关键词 | 这篇文章主攻哪个词？ |
| 辅助关键词 | 有哪些同义词、长尾词、FAQ？ |
| 搜索意图 | 用户现在是学习、比较、购买前调研，还是售后使用？ |
| 父级商业页 | 文章最终要导向哪个分类页/产品页/解决方案页？ |
| 文章类型 | 购买指南、对比文、教程文、价值解释文、规格文、场景方案、FAQ？ |
| 用户问题 | 用户搜索这个词真实想解决什么？ |
| 一句话答案 | 开头能不能直接回答用户问题？ |
| H2 结构 | 是否按用户决策顺序展开？ |
| 竞品缺口 | 前排文章缺什么？我们凭什么更好？ |
| 原创证据 | 图片、表格、数据、测试、案例、参数、经验是什么？ |
| 内链计划 | 链到哪些商业页和相关文章？ |
| CTA | ToB 是询盘/报价，ToC 是集合页/商品页/购买动作 |
| FAQ | 从 SERP、People Also Ask、Semrush、GSC 提取 |
| 复盘指标 | 发布后看排名、展现、点击、内链点击、转化辅助 |

## 7. 内容规划表的推荐结构

通用内容规划表建议包含这些字段：

| 字段 | 说明 |
|---|---|
| 层级 | L1/L2/L3，表示页面结构 |
| 关键词 | 原始关键词 |
| 主题簇 | 归属哪个产品线或问题域 |
| 搜索量 | Semrush/Ahrefs/GSC 数据 |
| CPC | 商业价值参考 |
| KD | 难度参考 |
| 搜索意图 | Informational / Commercial / Transactional / Mixed |
| SERP 获胜类型 | 前排是商品页、分类页、文章、论坛还是百科 |
| 推荐页面类型 | 分类页、产品页、博客、FAQ、暂不做 |
| 父级商业页 | 文章或子页面要支撑的商业页 |
| 目标 URL | 新建或已有 URL |
| 文章类型 | 如果写文章，选择文章类型 |
| 评分 | 按 100 分模型计算 |
| 优先级 | P0/P1/P2/P3 |
| 推荐动作 | 写文章、做页面、合并、更新、暂缓 |
| Brief 状态 | 未写、已写、待补证据、待发布 |
| 发布状态 | 未发布、已发布、待更新 |
| 复盘指标 | GSC 展现、点击、排名、转化辅助 |

## 8. 当前 Excel 标准包的使用方式

建议这样使用当前几张 Excel：

1. 用 `google_seo_keyword_scoring_template.xlsx` 做全站关键词评分和页面类型初筛。
2. 用 `google_seo_article_keyword_standard_framework.xlsx` 做文章关键词筛选、网站定位输入、AI 交接和文章 Brief。
3. 用 `google_seo_keyword_to_article_standard.md` 作为给其他 AI 或团队成员的文字版规则。

如果只给别人一张表，优先给增强版 `google_seo_article_keyword_standard_framework.xlsx`，因为它包含网站定位、标准骨架、AI Prompt、评分表和 Brief 模板。

## 9. 可复制给 AI 的标准提示词

你是 Google SEO 内容策略师。请不要直接开始写文章。你需要先根据网站定位、关键词数据和 SERP 页面类型，完成关键词到页面/文章的规划。

请按以下顺序工作：

1. 先读取网站类型、商业目标、核心产品、目标市场、成交页面、站点阶段和内容资源。
2. 对每个关键词判断搜索意图和 SERP 前 10 名页面类型。
3. 将关键词分流到分类页、集合页、产品页、解决方案页、博客文章、FAQ/段落或暂不做。
4. 按 100 分模型评分：搜索意图匹配20、商业价值20、需求强度15、难度机会15、页面可做性15、主题簇价值10、数据置信度5。
5. 只有适合文章承接的关键词，才进入博客规划。
6. 每篇文章必须指定父级商业页、文章类型、用户问题、H2结构、原创证据、内链、CTA 和复盘指标。
7. 输出关键词评分表、页面映射表、P0/P1 内容日历和文章 Brief。

注意：关键词不是都拿来写文章。博客必须服务商业页，ToB 服务询盘，ToC 服务加购和下单。

## 10. 最终判断

这套标准可以作为你以后所有独立站 SEO 项目的基础标准。

但每个新项目开始前，必须先替换三类变量：

1. 网站类型：ToB、ToC、SaaS、服务站。
2. 商业页类型：产品聚合页、分类页、集合页、商品页、解决方案页。
3. 转化目标：询盘、加购、下单、注册、下载。

只要这三类变量填对，后面的关键词评分和博客规划逻辑就可以复用。

## 外部资源引用规则

`References` 不是每篇文章都必须出现的固定模块，而是根据内容风险和事实来源触发的模块。不要每篇文章都机械复用同一组官方来源，否则会显得模板化，也会削弱内容可信度。

不需要强制引用的文章类型：

- 普通选购经验文。
- 口味解释文。
- 使用场景文。
- 产品灵感文。
- 只基于自有产品信息、站内数据、真实图片、客户问题、人工经验整理的文章。

需要引用的文章类型：

- 涉及健康、法律、年龄、合规、安全、电池、金融、医疗等高风险主题。
- 正文引用了官方定义、法规、统计数据、行业标准、平台规则或研究资料。
- 文章结论依赖外部事实，而不是自有经验或自有数据。

如果触发引用，正文末尾增加 `## References`，并使用描述性锚文本，不展示裸 URL，不使用 `source`、`click here`、`read more` 这类泛锚文本。

标准格式：

```markdown
## References

- [FDA overview of e-cigarettes, vapes, and other ENDS products](https://www.fda.gov/tobacco-products/products-ingredients-components/e-cigarettes-vapes-and-other-electronic-nicotine-delivery-systems-ends)
- [EPA guidance for used lithium-ion batteries](https://www.epa.gov/recycle/used-lithium-ion-batteries)
```

执行标准：

- 普通文章可以 0 个引用。
- 高风险或事实密集文章通常引用 2-5 个高质量来源即可。
- 每个来源必须对应正文中的具体事实，不要把同一套来源复制到所有文章。
- 锚文本优先写成“机构名称 + 页面主题”。
- 正文中可以自然提到来源，例如 `The FDA categorizes these products as...`，但完整来源链接统一放在文末 `References`。
- 如果文章没有使用外部来源，不要添加 `References` 区块。

## 11. 文章图片与插入位置标准

独立站文章配图不能只追求好看，必须服务阅读理解、商业承接和 SEO。后续所有文章生成或图片生成任务，都要明确图片具体放到文章哪个位置。

每篇文章如果生成图片，必须同步输出 `Image Placement Map`：

| 字段 | 要求 |
|---|---|
| Image | 图片用途，例如 cover、comparison、process、checklist、product detail、responsible use |
| File | 图片文件名或建议文件名 |
| Exact placement | 精确插入点，必须写清楚放在哪个 H2/H3、哪段后、哪个表格前后，或是否作为 CMS featured image |
| Purpose | 图片承担的阅读任务，例如解释概念、辅助对比、降低购买顾虑、强化合规提醒 |
| Alt tag | 描述性 alt，目标英文市场用英文，不堆关键词 |

通用放置规则：

- 封面图：作为 CMS featured image；如果没有 featured image 字段，则放在 H1 下方、正文第一段之前。
- 定义图：放在定义或概念解释段落后。
- 结构图/流程图/步骤图：放在对应的工作原理、教程、流程、使用方法章节内。
- 对比图：放在对比表之前或之后；有表格时优先放在表格前。
- 购买清单图：放在 buyer guide、what to check、checklist 章节开头或第一段后。
- 合规/安全/回收图：放在对应的 safety、compliance、disposal、responsible use 章节附近。
- 产品或 CTA 图：放在用户已经获得足够信息之后，靠近内链或 CTA，避免开头强推。

Alt 标签规则：

- 只描述图片事实，不写夸张营销词。
- 可以自然包含关键词，但不能关键词堆砌。
- 避免 `best`、`safe`、`healthy`、`guaranteed`、`cure`、`quit smoking` 等无法证明或高风险词。
- 教程图、对比图、结构图、检查表图属于正文信息图，必须写 alt；纯装饰图才可以空 alt。
