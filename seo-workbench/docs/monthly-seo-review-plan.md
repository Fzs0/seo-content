# 月度 SEO 复盘模块规划

来源：`@瑜东谷歌SEO_20260701_B2B独立站月度SEO复盘_排名流量询盘一起看_最....mp4`

说明：该视频没有字幕轨，本规划基于视频关键帧、可见字幕和演示页面结构总结。核心不是“继续多写文章”，而是把 GSC、当前排名、GA4、询盘/订单和页面承接统一成月度复盘，再输出下月 Action Plan。

## 核心方法

月报不是汇报，而是打法。复盘页必须先给结论，再给动作：

> 本月结论：先证明需求，再补强排名承接。

判断顺序：

1. 用 GSC 判断搜索需求是否成立：曝光、点击、核心查询词、目标国家。
2. 用当前排名工具校验承接：DataForSEO、Semrush Rank 或其他 SERP provider，而不是直接把 GSC 平均排名当实时排名。
3. 用 GA4 判断流量质量：active users、sessions、views、engagement、bounce、avg time、top GEO、landing pages、source / medium。
4. 用询盘、订单、加购或表单数据判断商业价值。
5. 最后输出动作：新写文章、更新旧文、补内链、规划产品页/集合页、技术修复或继续观察。

## 机会类型

| 类型 | 触发条件 | 动作 | 不要做什么 |
| --- | --- | --- | --- |
| Quick Win | 当前排名 11-30，或 GSC 有曝光且已有承接页 | 补内链、补 FAQ、补对比段落、优化标题摘要 | 不要另写一篇重复文章 |
| Blog Supporting | 产品/集合页需要解释型或导购型内容支撑 | 写支持文章，并规划到父级商业页的自然内链 | 不要让博客抢产品页主词 |
| Product Page Needed | GSC 有商业需求但没有合适产品页/集合页 | 规划产品页或集合页，再决定是否补博客 | 不要用博客硬接购买词 |
| Internal Link | 已有页面有曝光或停留，但排名/点击弱 | 从相关旧文、新文、集合页补内链 | 不要堆同一个锚文本 |
| Update Old Page | 旧文已有排名/曝光但内容薄、过期或 CTR 弱 | 重写标题、补内容、补图片/表格/FAQ | 不要开新文互相竞争 |
| Monitor | 有苗头但数据不足 | 加入观察列表，下月复查 | 不要过早投入生产资源 |

## 页面规划

### 数据源中心

把关键词、站点文章、GSC、GA4、当前排名、询盘/订单接到同一个站点画像。

| 数据源 | 状态 | 用途 |
| --- | --- | --- |
| Semrush 关键词池 | 已有导入 | 建立关键词池、主题簇、初始意图和难度 |
| 站点文章 API | 已有 WordPress、自建博客、主站 OpenAPI | 判断已覆盖主题、重复竞争、内链机会、旧文优化对象 |
| GSC | 先 CSV，后 API | 证明本站真实搜索需求 |
| 当前排名 | 预留 DataForSEO / Semrush Rank / SerpAPI | 校验当前排名和排名 URL |
| GA4 | 先 CSV，后 Data API | 判断流量质量、入口页质量和目标国家 |
| 询盘/订单/加购 | 后续接表单、Shopify、CRM 或自建接口 | 判断 SEO 是否有商业价值 |

### 月度 SEO 复盘页

目标：回答“这个月 SEO 有没有价值，下个月该做什么”。

| 模块 | 重点 |
| --- | --- |
| 月度结论卡 | 本月结论、需求是否成立、承接是否不足、下月主打法 |
| GSC KPI | clicks、impressions、CTR、avg position、核心查询词 |
| 当前排名校验 | 当前排名位置、排名 URL、URL 是否匹配、页面类型 |
| GA4 质量 | active users、sessions、engagement、bounce、avg time、top GEO |
| 商业价值 | 询盘、订单、加购、表单来源、归因页面 |
| 风险提醒 | 高曝光 0 点击、当前未排名、页面类型错配、国家/语言错配、无承接页 |

### GSC + 当前排名校验页

| 字段 | 说明 |
| --- | --- |
| 查询词 | GSC query 或关键词池词 |
| 主题集群 | 对应产品线/主题 |
| 目标市场 | 国家/语言 |
| GSC clicks / impressions / CTR / avg position | 需求和历史信号 |
| 当前排名 provider | DataForSEO、Semrush Rank 等 |
| 当前排名 / 当前排名 URL | 当前 SERP 真实承接 |
| 预期承接 URL | 目标页 |
| URL 是否匹配 | 匹配、不匹配、待确认 |
| 当前 URL 类型 / 预期页面类型 | 博客、产品页、集合页、支柱页、FAQ |
| 机会类型 | Quick Win、Product Page Needed 等 |
| 优先级 | P0/P1/P2/P3 |
| 建议动作 | 回流到内容中枢、页面规划、内链任务或待办 |

### GA4 质量分析页

目标：判断流量值不值钱，而不是只看访问量。

| 字段 | 说明 |
| --- | --- |
| 入口页 | landing page |
| 国家 | country |
| 来源 | source / medium |
| Active Users / Sessions / Views | 基础规模 |
| Engagement Rate / Bounce Rate / Avg Time | 质量信号 |
| 质量分 | 可用公式做辅助评分 |
| 建议动作 | 强化 CTA、补内链、更新内容、观察、暂缓 |

## 开发顺序

1. 先做数据源中心和月度复盘壳子，支持 GSC/GA4 CSV 手动导入。
2. 用 GSC + 关键词池 + 文章库存生成 Action Plan。
3. 把 Action Plan 推入内容中枢、生产、页面规划和内链任务。
4. 再接 GA4 Data API、GSC API、当前排名 provider、询盘/订单接口。
5. 最后做月报导出：Markdown、HTML、CSV，后续再扩展 PDF。

## 对当前项目的结论

当前系统已经有关键词池、内容中枢、AI 生成、WordPress/自建博客/主站发布接口。下一步不要继续只堆写作功能，而应增加“复盘层”：

1. 先判断哪些站点、国家、页面真的有需求。
2. 再判断哪些关键词是排名承接问题。
3. 再判断该新写、改旧文、做内链、做产品页，还是观察。
4. 最后把任务推入内容生产，而不是从关键词直接写文章。
